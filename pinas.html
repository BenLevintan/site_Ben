Pi-NAS Container Storage System

A custom Network Attached Storage (NAS) solution built from scratch using a Raspberry Pi 5, designed specifically for container storage and development workflows.

Project Overview

This project involved building a complete NAS solution from the ground up, starting with hardware selection and ending with a fully configured system optimized for container storage. The system serves as both a development environment and a reliable storage solution for my projects.

Hardware Specifications
  • Main Board: Raspberry Pi 5 (4GB RAM)
  • Storage: 2TB SATA SSD (Primary) + 120GB SATA SSD (Secondary)
  • Case: 3D printed four-bay NAS case (dark marine green)
  • Cooling: Active cooling solution with temperature monitoring
  • Network: Native Gigabit Ethernet port on Pi 5
  • Power: 5.25mm barrel jack 12V/5A power supply
  • SATA HAT: Penta SATA HAT by RADXA
Hardware Components
Software Configuration
  • OS: Debian Lite 64-bit
  • Container Runtime: Docker with Docker Compose
  • File System: Btrfs for snapshots and data integrity
  • Network Services: Samba, NFS, and SSH
  • Monitoring: Prometheus + Grafana stack
  • Backup: Automated rsync scripts with encryption
Core Technologies
Running Containers
  • Jellyfin: Media server for streaming and organizing multimedia content
  • Tailscale: Secure mesh VPN for remote access and networking
  • OpenMediaVault: Web-based network attached storage (NAS) solution for easy management
Container Services
Key Features
Container-Optimized Storage: Configured with Docker volumes and bind mounts for seamless container data persistence.
High Availability: RAID-like redundancy with automated backup and recovery procedures.
Network Performance: Optimized for gigabit speeds with low-latency access to container data.
Remote Management: Full SSH access with web-based monitoring and management interfaces.
Completed Build
Technical Challenges

The most challenging aspects included configuring the storage subsystem for optimal performance, setting up proper cooling management for sustained workloads, and implementing a robust backup strategy that wouldn't impact container performance. The system now runs 24/7 with excellent stability and performance.
